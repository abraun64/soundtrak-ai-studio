# Applied marketing frameworks

**What this is.** Three reference catalogues in one file — **buyer psychology & mental models**, a **conversion (CRO) page-audit framework**, and a **voice-of-customer research method**. All generic and tenant-agnostic: the *applied toolkit* the Creative Director (and, via CM's Per-Step-Brief slice, the Producer and Insights Manager) pull on demand. Kept as one document with clear section anchors so an agent pulls only the section it needs — not the whole file.

**How it's accessed (pull the relevant section, don't load it all):**
- **§1 Mental models** — CD when shaping a concept's persuasion; Producer (via CM slice) for copy/page psychology.
- **§2 Conversion audit** — Producer on any converting page; CD when the concept *is* a landing/pricing/feature page.
- **§3 Voice of customer** — Insights Manager (Phase 1) + the Brief grill-me interview; CD to ground the concept's audience insight.

**Relationship to the Soundtrak Playbook** (`Soundtrak_Playbook.md`): the Playbook is the tenant's strategic **doctrine** (positioning, GTM, the consulting principles — *how Soundtrak thinks*); this file is the generic applied **toolkit** (*how anyone executes*). Where a model appears in both (e.g. Jobs-to-Be-Done), the **Playbook owns the strategic principle and this file owns the applied toolkit — cross-link, don't duplicate.**

**Source.** Mined (re-expressed, tenant-agnostic, attributed) from the `marketing-psychology`, `cro`, and `customer-research` skills in [coreyhaines31/marketingskills](https://github.com/coreyhaines31/marketingskills) (MIT). Standard behavioural-science + conversion canon; near-static reference — maintain lightly, one place.

# §1 — Mental models

*~60 thinking tools for why people buy and how to influence behaviour ethically — a lookup catalogue the CD/Producer cite when shaping a concept, page, offer, or sequence. A concept is stronger when it names the model it applies or tests (e.g. "the hero uses the contrast effect — show the before-state plainly so the after lands").*

**How to use it.** (1) Name the behaviour you're trying to change. (2) Pick the 1–3 models that fit the friction. (3) Apply them, and (4) only where the claim is genuinely true — scarcity you invent, social proof you don't have, or a decoy that misleads all erode trust and, in regulated tenants, invite compliance risk. Steal the *model*, not a manipulation.

## 1.1 Foundational thinking models
*Sharpen strategy; make sure you're solving the right problem before optimising.*

| Model | Principle | Apply |
|---|---|---|
| First principles | Reason up from basic truths, not by copying rivals | Ask *why* this channel/asset before building it; 5 Whys to the real job |
| Jobs to Be Done | People "hire" a product for an outcome, not for features | Frame around the hole, not the drill — the outcome the buyer wants |
| Circle of competence | Play where you have genuine edge | Double down on channels where the tenant has real advantage; don't chase all |
| Inversion | Ask "what guarantees failure?" then avoid it | List everything that would sink the campaign, prevent each |
| Occam's razor | Simplest explanation usually wins | Check the obvious (broken form, page speed) before complex attribution |
| Pareto (80/20) | ~80% of results from ~20% of effort | Find the vital few channels/assets/segments; cut the rest |
| Local vs global optima | Best nearby ≠ best overall | Don't perfect subject lines if email is the wrong channel — zoom out first |
| Theory of constraints | One bottleneck caps throughput | Fix the binding constraint (traffic? conversion? retention?) before optimising elsewhere |
| Opportunity cost | Every yes is a no to something else | Judge low-ROI effort against the high-ROI thing it displaces |
| Diminishing returns | Extra investment yields progressively less | Know when to diversify rather than double down |
| Second-order thinking | Consider effects of the effects | A discount lifts revenue now (1st) but trains discount-waiting (2nd) |
| Map ≠ territory | The model isn't the reality | The dashboard/persona is a useful model; stay close to real users |
| Probabilistic thinking | Think in likelihoods, plan for ranges | Don't stake everything on one campaign; plan for underperformance |
| Barbell strategy | Pair proven safety with small wild bets; avoid the mediocre middle | ~80% into proven channels, ~20% into experiments |

## 1.2 Buyer psychology
*How customers actually think, decide, and behave.*

| Model | Principle | Apply |
|---|---|---|
| Fundamental attribution error | We blame character over circumstance | When buyers don't convert, examine your process before blaming them |
| Mere exposure | Familiarity breeds liking | Consistent presence across touchpoints builds preference over time |
| Availability heuristic | Vivid/recent examples feel more likely | Case studies + testimonials make success easy to imagine |
| Confirmation bias | People seek belief-confirming info | Align with what the audience already believes; fighting head-on rarely works |
| Lindy effect | The long-lived is likely to persist | Trust fundamentals (clear value, social proof) over trendy tactics |
| Mimetic desire | Wanting is socially contagious | Show desirable people wanting it; waitlists/exclusivity trigger it |
| Sunk-cost fallacy | Past spend irrationally justifies more | Kill underperforming campaigns; sunk spend ≠ future spend |
| Endowment effect | We value what we "own" more | Free trials/samples let buyers own it, then dread giving it up |
| IKEA effect | We value what we helped build | Let buyers configure/customise; effort raises perceived value |
| Zero-price effect | "Free" is a category, not a low price | Free tier/trial/shipping has outsized pull; $1→$0 > $2→$1 |
| Present bias / hyperbolic discounting | Immediate rewards beat future ones | Emphasise "today" benefits over "ROI in 6 months" |
| Status-quo bias | The current state feels safe | Make switching feel safe and one-click ("import in one click") |
| Default effect | Pre-selected options get accepted | Pre-select the recommended plan (ethically) |
| Paradox of choice | Too many options paralyse | Three tiers beat seven; recommend one "best for most" |
| Goal-gradient | Effort accelerates near the goal | Progress bars, "80% done", almost-there nudges |
| Peak-end rule | We judge by the peak and the end | Design a memorable peak + a strong ending (thank-you, follow-up) |
| Zeigarnik effect | Open loops nag until closed | "You're 80% done", abandoned-cart, cliffhangers |
| Pratfall effect | A small flaw makes the competent more likeable | "We're not the cheapest, but…" can raise trust |
| Curse of knowledge | Experts can't un-know; they over-assume | Test copy on people new to the space; it's less obvious than you think |
| Mental accounting | Money is bucketed, not fungible | "$3/day" lands differently from "$90/month" |
| Regret aversion | We avoid choices that might cause regret | Guarantees, free trials, "no commitment" defuse regret-fear |
| Bandwagon / social proof | We follow the crowd; popularity signals safety | Counts, logos, reviews, "trending" build confidence |

## 1.3 Persuasion & influence
*Ethical influence on a decision.*

| Model | Principle | Apply |
|---|---|---|
| Reciprocity | Give first, people want to return it | Free content/tools/tier before asking for anything |
| Commitment & consistency | Small yes → bigger yes | Ladder small commitments (signup → trial → plan) |
| Authority | We defer to credible experts | Expert endorsements, certifications, "featured in", thought leadership |
| Liking / similarity | We say yes to those like us | Relatable spokespeople, founder story, "built by X for X" |
| Unity | Shared identity is powerful — "one of us" | Insider language, shared values, tribe positioning |
| Scarcity / urgency | Limited availability raises perceived value | Genuine limited-time/low-stock/exclusive access only |
| Foot-in-the-door | Small request first, then escalate | Trial → paid → annual → enterprise |
| Door-in-the-face | Big ask first, retreat to the real one | Show enterprise pricing, then the affordable starter |
| Loss aversion / prospect theory | Losses hurt ~2× the pleasure of equal gains | Frame the cost of *not* acting ("don't miss out") |
| Anchoring | The first number frames the rest | Show the higher price/competitor/enterprise tier first |
| Decoy effect | A worse third option flatters your target | A clearly-inferior tier makes the preferred one obvious |
| Framing | Same facts, different frame, different feel | "90% success" > "10% failure"; frame positively |
| Contrast | Things read relative to their comparison | Show the before-state plainly so the after feels vivid |

## 1.4 Pricing psychology

| Model | Principle | Apply |
|---|---|---|
| Charm pricing / left-digit | $99 reads far below $100 | .99/.95 endings for value-positioned products |
| Rounded-price (fluency) | Round numbers feel premium, process easier | Round prices for premium; charm prices for value |
| Rule of 100 | <$100 → % off feels bigger; >$100 → $ off feels bigger | $80: "20% off" > "$16 off"; $500: "$100 off" > "20% off" |
| Good-better-best relativity | Price is judged against shown options | Three tiers; the dear one anchors, the cheap one floors, the middle wins |
| Mental accounting (pricing) | Reframing the same price changes perception | "$1/day" / "less than a coffee" |

## 1.5 Design & delivery models
*Systems that reliably produce action.*

| Model | Principle | Apply |
|---|---|---|
| Hick's law | More/complex choices = slower decisions, more abandonment | One clear CTA; fewer form fields |
| AIDA | Attention → Interest → Desire → Action | Sequence pages/campaigns through the stages in order |
| Rule of 7 | ~7 touchpoints before conversion | Multi-touch: retargeting + email + consistent presence compound |
| Nudge / choice architecture | Presentation shapes choice without restricting it | Defaults, ordering, friction reduction |
| BJ Fogg (B=MAP) | Behaviour = Motivation × Ability × Prompt | All three must coincide; design for the weakest |
| EAST | Make it Easy, Attractive, Social, Timely | Reduce friction, make it appealing, show others, ask at the right moment |
| COM-B | Behaviour needs Capability, Opportunity, Motivation | Can they? Is the path clear? Do they want to? Address all three |
| Activation energy | The cost of *starting* blocks easy tasks | Pre-fill, templates, quick wins — make step one trivial |
| North Star metric | One metric captures delivered value | Align every effort to it |
| Cobra effect | Incentives can backfire | Test incentives — a referral bonus can attract gamers |

## 1.6 Growth & scaling models

| Model | Principle | Apply |
|---|---|---|
| Feedback loops | Output becomes input | Build virtuous loops (users → content → SEO → users) |
| Compounding | Small consistent gains accumulate; early gains matter most | Start content/SEO/brand early; benefits accrue exponentially |
| Network effects | More users → more value | Shared workspaces, integrations, marketplaces, communities |
| Flywheel | Sustained effort → self-sustaining momentum | Content → traffic → leads → customers → case studies → content |
| Switching costs | The cost of leaving creates retention | Ethically raise them: integrations, accumulated data, workflow fit |
| Exploration vs exploitation | Balance new bets with optimising what works | Keep working channels; ring-fence budget for experiments |
| Critical mass / tipping point | A threshold makes growth self-sustaining | Reach depth in one segment before breadth |
| Survivorship bias | Winners are visible; failures aren't | Study failed campaigns too — the viral hit had 99 invisible flops |

## 1.7 Quick reference — challenge → models

| Challenge | Reach for |
|---|---|
| Low conversions | Hick's law · activation energy · BJ Fogg · friction audit |
| Price objections | Anchoring · framing · mental accounting · loss aversion |
| Building trust | Authority · social proof · reciprocity · pratfall |
| Increasing urgency | Scarcity · loss aversion · Zeigarnik |
| Retention / churn | Endowment · switching costs · status-quo bias |
| Growth stalling | Theory of constraints · local vs global optima · compounding |
| Decision paralysis | Paradox of choice · default effect · nudge |
| Onboarding | Goal-gradient · IKEA effect · commitment & consistency |

**Diagnostic questions**: What specific behaviour am I changing? What does the audience believe *before* they meet this? Where in the journey is it? What's actually preventing the action? Has it been checked against real customers (§3)?

# §2 — Conversion audit (CRO)

*A diagnostic lens for any page whose job is to convert — landing, homepage, pricing, feature, signup form. Use it two ways: (1) build a converting page **to** this checklist; (2) it's an audit grid for critiquing a draft (or a live page) before it ships. Pairs with `craft/copywriting.md` (the writing) and §1 (the psychology behind each lever).*

**First, establish context** — a page can only be judged against its job:
- **Page type** — homepage · landing · pricing · feature · blog · about.
- **Primary conversion goal** — one action (sign up · book demo · purchase · subscribe · download · contact sales).
- **Traffic context** — where the visitor came from (organic · paid · email · social) and what they already know.

## 2.1 The seven dimensions (in impact order)

1. **Value-proposition clarity** *(highest impact)* — In 5 seconds, does the visitor grasp what this is and why they should care? Is the primary benefit specific, differentiated, and in the customer's words? *Failure modes:* feature-focused not benefit-focused · too clever, sacrificing clarity · trying to say everything.
2. **Headline effectiveness** — Does it carry the core value prop, specifically, and match the traffic source's message? *Strong patterns:* outcome-led ("Get [outcome] without [pain]") · concrete numbers/timeframes · social proof ("Join 10,000+ teams…").
3. **CTA placement, copy & hierarchy** — One clear primary action, visible without scrolling, with value-carrying button copy ("Start free trial" / "Get my report", not "Submit"). Logical primary vs secondary; repeated at decision points.
4. **Visual hierarchy & scannability** — Can a scanner get the message? Are the important elements prominent? Enough whitespace? Do images support rather than distract?
5. **Trust signals & social proof** — Customer logos (recognisable), specific attributed testimonials (with photos), case-study numbers, review scores/counts, security badges where relevant. *Placement:* beside CTAs and right after benefit claims.
6. **Objection handling** — Address price/value, "will it work for *my* situation?", implementation difficulty, "what if it fails?" — via FAQ, guarantees, comparisons, process transparency.
7. **Friction points** — Too many form fields · unclear next step · confusing nav · needless required fields · mobile issues · slow load.

## 2.2 Output shape
Return recommendations in this order (also the CM/operator-facing shape):
- **Quick wins** — easy changes, likely immediate impact.
- **High-impact changes** — bigger lifts worth the effort, prioritised.
- **Test ideas** — hypotheses to A/B rather than assume.
- **Copy alternatives** — for headline + CTA, 2–3 options each with rationale.

## 2.3 Page-specific emphases
| Page | What matters most |
|---|---|
| Homepage | Positioning for cold visitors · quick path to the common conversion · serve both "ready to buy" and "still researching" |
| Landing | Message-match with the traffic source · single CTA (drop nav if possible) · the whole argument on one page |
| Pricing | Clear plan comparison · a recommended plan indicated · defuse "which plan is right for me?" |
| Feature | Connect each feature to a benefit · use cases/examples · clear path to try/buy |
| Blog | Contextual CTAs matching the topic · inline CTAs at natural stopping points |

## 2.4 Form optimisation (when the conversion is a form)
Cut fields to the minimum that's genuinely required · consider multi-step for long forms (progress + goal-gradient) · inline, human error handling · never require what you don't need · label the value of submitting, not the act.

**Diagnostic questions**: Current conversion rate and goal? · Traffic source? · What does the flow *after* this page look like? · Any research/heatmaps/session recordings? · What's already been tried?

# §3 — Voice of customer

*A method for grounding positioning, concepts, and copy in what customers actually say — verbatim — rather than in assumption. Feeds the Brief (audience insight), the CD's concept (the tension it dramatises), and the Producer's copy. Complements the Insights Manager's evidence sweep + `tenant/research-library/`; reinforces the Brief grill-me interview and the `content-subedit` "customer language over company language" rule.*

**The one rule that carries the rest:** capture **exact quotes, not paraphrases.** *"We were drowning in spreadsheets"* is worth more than *"manual-process inefficiency"* — it's the copy, pre-written, in the buyer's own mouth.

## 3.1 Mode 1 — analyse existing assets
| Asset | Mine for |
|---|---|
| Interview / sales-call transcripts | Pains, triggers, desired outcomes, objections, alternatives; the *moment* they decided to look; what they tried before |
| Survey results | Segment before concluding; flag where open-ended answers contradict multiple-choice; the 20% of responses with the real signal |
| Support conversations | Recurring complaints, confusion points, "I wish it could…"; categorise first (bug vs confusion vs missing feature vs expectation mismatch) |
| Win/loss & churn notes | Wins: what tipped it, what nearly lost it. Losses/churn: price vs features vs fit vs timing — segment by reason, don't average |
| NPS | Passives/detractors > promoters for improvement; always pair the score with the verbatim |

**Extraction fields (per asset):** (1) Jobs to Be Done — functional (task) · emotional (how they want to feel) · social (how they want to be seen); (2) Pain points — prioritise *unprompted* + emotional; (3) Trigger events — what changed and made them look; (4) Desired outcomes — success in their words (exact quotes); (5) Language & vocabulary — the exact phrases (copy gold); (6) Alternatives considered — including do-nothing, hire, build.

**Synthesis:** cluster by theme → score each by **frequency × intensity** → segment by profile → pull 5–10 **money quotes** per theme → flag contradictions.

## 3.2 Mode 2 — digital watering holes
| ICP | Primary sources |
|---|---|
| B2B SaaS / technical | Reddit (role subs), G2/Capterra, Hacker News, LinkedIn, Indie Hackers, SparkToro |
| SMB / founders | Reddit (r/entrepreneur, r/smallbusiness), Indie Hackers, Product Hunt, Facebook Groups |
| Developer / DevOps | r/devops, r/programming, Hacker News, Stack Overflow, Discord |
| B2C / consumer | App-store reviews (1–3★), hobby/lifestyle subs, YouTube/TikTok/IG comments |
| Enterprise | LinkedIn, analyst reports, G2 Enterprise filter, job postings, SparkToro |

*Quick guide:* have a category? → G2/Capterra (yours + competitors). Raw language? → Reddit + YouTube comments. Trigger events? → LinkedIn posts, job postings, "Ask HN". Where they gather? → SparkToro. Capture per item: source (platform · URL · date) · verbatim quote · context · sentiment · theme tag · profile signals.

## 3.3 Quality guardrails
- **Confidence label on every insight** — **High** (3+ independent sources, unprompted, consistent) · **Medium** (2 sources, or prompted, or one segment) · **Low** (single source; validate). Mirrors the Insight Brief's evidence discipline.
- **Recency** — weight the last 12 months; an old transcript may reflect a different product and buyer.
- **Sample bias** — reviewers skew power-user/strong-opinion; tickets skew to problems; Reddit skews technical/skeptical. Factor it in before saying "all customers".
- **Minimum viable sample** — no personas/messaging conclusions from fewer than 5 independent data points per segment.

## 3.4 Personas (built, never invented)
Only after 5–10 data points from a consistent segment. Structure: profile (title range · size · industry · reports-to) · primary JTBD (one sentence) · trigger events · top pains (their words) · desired outcomes (incl. how it makes them look) · objections/fears · alternatives · key vocabulary (sourced quotes) · how to reach them. **Anti-patterns:** cute names unless useful · averaging across segments · inventing details (leave blank) · never refreshing (personas decay — revisit quarterly).

## 3.5 Deliverables to offer
Research-synthesis report · VOC quote bank (verbatims by theme, for copy) · persona doc (1–3) · JTBD map (functional/emotional/social by segment) · competitive-intel summary · research-gap analysis.
