Deployment guide

Get your own copy running on your computer. No coding, and no experience with Claude needed. A few quick installs, then the system sets itself up — about 30–45 minutes, most of it waiting on downloads.

Never used Claude before? Read this first. Your system runs inside Claude Code — an AI assistant that lives on your computer. You use it by typing plain‑English requests (like messaging a very capable colleague); it does the work and shows you — you never write code yourself. Three things to know up front:
  • You'll need a Claude subscription — a Claude Pro or Max plan (the same account that runs Claude.ai). It isn't free.
  • You type your requests and read its replies in a window (a "terminal"). Don't be put off by the word — you only ever type normal sentences.
  • Before it ever changes a file or runs anything, it shows you and asks permission. Nothing happens without your OK.
This guide installs everything once. After that, you just chat.
Don't want to do the setup yourself? Whoever gave you this system can do steps 1–4 for you on a screen-share in about half an hour. You only ever need the chat (step 5 onward).
Step 1 — Install Python

This runs the system's behind-the-scenes scripts.

  1. Go to python.org/downloads and click the big yellow Download Python button.
  2. Open the file you just downloaded.
  3. IMPORTANT: on the first screen, tick "Add Python to PATH" (bottom of the window) before clicking Install Now. This one tick prevents the most common setup problem.
  4. Let it finish, then click Close.
Step 2 — Install Claude Code & sign in

Claude Code is the app you'll actually use — the AI you type your requests to.

  1. Get a Claude account first. You need a paid plan — Claude Pro or Max (the same login as Claude.ai). If you don't have one, sign up at claude.com/pricing.
  2. Install Claude Code using Anthropic's official guide for Windows: code.claude.com/docs/en/setup.
  3. Do the 5‑minute quickstart — it shows a first‑timer exactly what to click: code.claude.com/docs/en/quickstart. The first time you open Claude Code it opens your browser to log in — sign in with your Pro/Max account. You only do this once.
Step 3 — Download the system from GitHub

The whole system is one folder of files, hosted on GitHub. You don't need a GitHub account — anyone can download it.

  1. Open github.com/abraun64/soundtrak-ai-marketing-system.
  2. Click the green < > Code button (top‑right of the file list), then Download ZIP.
  3. The ZIP saves into your Downloads folder. Right‑click it → Extract All… → Extract.
  4. Move the extracted folder into your OneDrive (or Dropbox) folder, then rename it Marketing AI System. This step matters: keeping it inside OneDrive/Dropbox gives you an automatic offsite backup of everything you do, with no extra setup.
Where it should end up: something like C:\Users\<you>\OneDrive\Marketing AI System. That one folder is your system — everything lives inside it.
Step 4 — Open your system in Claude Code

Claude Code always works inside one folder. You point it at your Marketing AI System folder so it knows where your system lives.

  1. Open Claude Code.
  2. Open the Marketing AI System folder as your project — in the desktop app choose Open folder and pick it; if you use the terminal version, open a terminal inside that folder and run claude.
  3. You're in the right place when Claude Code shows Marketing AI System as the current project/folder.
How you "talk to it": type your request in the box at the bottom and press Enter. It replies and gets to work. Type / on its own to see the list of built‑in commands and skills (that's what something like /system-manager is — you pick it from that list). Whenever it wants to change a file or run something, it shows you first and asks — approve it and it carries on. Nothing happens without your OK.
What's a "chat"? Each conversation in Claude Code is its own chat session. You don't have to name them, but it helps a lot to name each chat for its job — e.g. Brand setup, Q3 launch, Retro — and to start a fresh chat for each new job (each campaign, each retro). That keeps each conversation focused and easy to find again.
Step 5 — Finish the install

The system isn't a wizard with buttons — you run it by typing plain‑English requests in a chat, and it asks you questions back and does the work.

In a chat, type:

set yourself up

It installs the last few pieces, checks everything works, and turns on automatic backup. If something's missing it tells you the one thing to fix, in plain language. When it says READY, setup is done. (One time only.)

One‑time licence agreement: the first time, setup shows you the licence and disclaimer and asks you to agree before it continues. The system is free for your own business use and is provided "as is", entirely at your own risk (full terms are in the LICENSE file). The assistant will show you the terms and ask you to reply I AGREE — do that once and it carries on.
Step 6 — Set up your working chats

Now set up the chats you'll use day to day. For each, press + New Session and make sure it's under the Soundtrak AI Studio project (connected to your folder).

  1. Improve the system — this chat is to make the system itself better and improve how it operates (it is not for running a campaign). Type "/system-manager" to set it up. Nothing is required from you on Day 0, but from here you can say "run a system retro", "capture an idea: …", or "add to backlog: …". This is how the system gets better over time — your ideas become tracked work you can pick up later.
  2. Brand setup — do this once, before anything else. This foundation is required before any campaign can start — it's the "brand brain" every campaign reuses, so you never re‑explain yourself. It collects things like your style guide, tone of voice, governance rules, target market and tech stack. Type "Brand setup" and the system interviews you and builds your foundation. Have these ready to paste in or describe:
    • Brand style guide — or just your logo, colours and fonts.
    • Tone of voice — how you sound, words you use and words you avoid.
    • Positioning & segments — what you stand for, and who you're selling to.
    • Rules you must follow — only if you're regulated (finance, health…): disclaimers, things you can't claim.
    • Where you publish + your martech — your channels, plus your tools (CRM, email, analytics). If any needs a password or key, the system shows you where to enter it safely — never in a file.
    • Best-practice examples — a few standout pieces for the system to learn from.
The one habit that matters: a fresh chat for each new job — each campaign, each retro — and give it a name. Everything you do is saved automatically and carried across chats through your foundation, so a new chat never loses your brand or your past work.
Step 7 — Your first campaign

That's it — you're set up. You're now ready for your first campaign: press + New Session (name it for the campaign, e.g. "Q3 launch") and type "start a campaign: <what it's for>". The Operator guide walks you through it, stage by stage.

Optional — make the gallery buttons open natively

In the asset gallery, the 📁 Open folder and ✏️ Edit copy buttons open in your browser by default. To make them open File Explorer / your text editor instead, run this one-time setup (Windows, no admin rights) from your project folder:

powershell -ExecutionPolicy Bypass -File ".\.claude\skills\asset-gallery\protocol\setup-protocol.ps1"

It auto-detects your install path — nothing to edit. Full details + how to undo are in .claude/skills/asset-gallery/protocol/setup-cookbook.md. Skip it and the buttons still work, just in the browser.

If the health check shows red

Step 4 runs a "doctor" that checks your setup. A red [FAIL] line always prints the exact fix. The usual culprits:

Red lineWhat it meansFix
python module: … MISSINGa helper library didn't installtype "set yourself up" again — it retries

"python is not recognised"Python installed without "Add to PATH"re-run the Step 1 installer and tick "Add Python to PATH"

Do I need a GitHub account?

No — not to download the system, and not to use it day to day. GitHub shows up in two separate ways, and only the optional one needs an account of your own:

GitHub touchpointWhose it isAccount needed?
The public page you download from (Step 3B)the people who make the systemNo — anyone can download
An optional online backup of your own workyours (a free private repo)Yes — but only if you want it

Backup & upgrades (the short version)

  • Backup is automatic once set up (Step 4): your work saves at the end of every session. The safety net is that local save plus your OneDrive/Dropbox folder — an offsite copy with zero extra setup. That's enough for most people, no GitHub account required.
  • Want a separate online backup in your own account? Optional — create a free private repo, then say "set yourself up" again and the system connects it.
  • Upgrades: your copy is yours and keeps working as-is — you never have to upgrade. If a newer version is released, whoever gave you the system can hand you the update.
