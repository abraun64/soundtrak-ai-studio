## Strategic spine
**Audience**: content leaders, CMOs, founders running (or trying to run) a content programme. **Big idea**: most teams put creation first and distribution last, so the work finishes before anyone has thought about where it goes — flip the sequence. **Key message**: the distribution plan should be built alongside the content plan, because the distribution plan should *shape* the content. **Proof**: Netwealth's one 50-page AdviceTech report → keynote at 10+ roadshows (2,000+ attendees), a buyers' guide (1,000 downloads/yr), 30+ articles, a tech newsletter (8,500/edition), internal book clubs, proactive media outreach. **Channel**: Substack anchor article + LinkedIn post + the Register B illustration.

## Creative move
**What it did (two transferable things — this is a Hybrid entry):**

**(1) The argument structure — anchor-to-derivatives.** A long-form piece built on a single visual-able idea: one anchor asset (the report) deployed as ~20 derivatives is the *same investment, differently deployed*. The "50:1" / "10:1" rule is the spine; the Netwealth example is the evidence; the "if you apply one thing" list makes it executable.

**The move**:
- Concrete opening scene (five-month research project, "now what?"), not a thesis announcement.
- Name the default model ("creation-first"), then break it — the master contrast register at article length.
- A single quotable line carries the argument: "Distribution isn't the thing that happens after the content is done. It's the constraint that should shape it before a word is written."
- Evidence embedded in the argument (Netwealth derivatives listed with real numbers), not cordoned into a case-study box.
- "If you apply one thing from this 🛠️" → an executable checklist (plan 10–20 derivatives before creation; atomise by format; design each derivative for its native channel; budget distribution as a line item equal to production).
- Closes on the implication, not a recap: "is it possible to exhaust your distribution capacity before you exhaust your creation capacity?"

**(2) The Register B image-prompt template.** Image Prompt A ("THE 50:1") is the **canonical boilerplate for every future Signal illustration** — the Behavior Gap prompt template. It encodes: Behavior Navy (`#1a2a5e`) felt-tip lines on pure white; ONE Signal Red (`#e63c3c`) element carrying the argument (here, the connecting arrow); strictly 2D, no shading/gradients/shadows; confident-but-imperfect hand-drawn lines; hand-lettered block capitals, max 3 labels; navy title at top (never red); no bottom text; the "squint test." Reuse this prompt structure verbatim, swapping only the visual concept + the single red element.

**Why it worked**:
- The 50:1 idea is inherently visual — it earns a single sketch that passes the squint test, which is the whole Register B discipline.
- "Same investment, differently deployed" reframes distribution from cost to leverage in one line.
- The checklist converts a POV into something a reader can action Monday morning.

**When to steal**:
- **Argument**: any long-form on content strategy, repurposing, owned media, or programme design — use the anchor-to-derivatives spine and the concrete-scene opener.
- **Prompt template**: ANY Signal illustration brief, or any tenant asset wanting the Behavior Gap register — start from Image Prompt A and change only the concept + the one red element.

**Anti-patterns / risks**:
- Atomisation without native-format craft = "a LinkedIn article that's just a shortened white paper." Each derivative must be made *for* its channel.
- The prompt template breaks if more than one element gets red (the two-zone carve-out aside) or if the title is rendered red.
- Listing derivatives without the investment-equality point misses the argument — distribution must be budgeted as a line item equal to production.
