Marketing Tactics — Master Reference

Exhaustive menu of marketing tactics grouped by theme, with a named-plays layer up top for the higher-order strategic mechanics that combine them. The Creative Director draws on this at Phase 2 when shaping each concept's channel mix and tactical mechanics; the Campaign Manager uses it at Phase 3 to compose the Plan's asset list, and slices relevant tactic definitions into Per-Step Briefs at Phase 4a. (Updated 2026-06-16 — see changelog at bottom.)

This is the menu, not a script. Picking tactics is a strategic call — fit to audience, channel, effort tier, and phase goal. Tactics that don't earn their place don't go in the plan.

Channel column distinguishes: - Owned — channels you control (your site, your email list, your social profiles) - Earned — coverage you don't pay for (press, awards, organic mentions) - Paid — channels where reach costs money (search ads, social ads, sponsorship)

Named plays — strategic mechanics (start here)

The tables below are a menu of tactic types. These named plays are the higher-order layer: each combines several menu tactics into one strategic move with a specific job-to-be-done. Reach for a play when its condition matches, then assemble it from the menu rows it cites. Worked external precedents live in tenant/library/. (Added 2026-06-16 from the operator's SelfWealth/Syfe strategy deck — patterns generalised, not the client specifics.)

Friction-reduction utility — "the cost of staying" calculator

  • Mechanic: an interactive tool that takes the user's current situation and returns one personalised, specific number that quantifies the cost of not acting — "you left $847 on the table last year," or "this migration takes 90 seconds, not a weekend." Pair the honest calculation with a one-click next step.
  • Job: beat inertia, not ignorance. In considered categories the barrier is rarely awareness — it's that the status quo requires no effort and the gain feels marginal. This play amplifies the pain of staying (a concrete dollar figure) while lowering the friction of switching (it does the hard part for them).
  • Reach for it when: an incumbent-dominated, high-switching-cost market where the prospect already half-suspects they're overpaying.
  • Assembles: Templates & tools · Landing page optimisation (CRO) · Customer journey mapping.

Proprietary-data content engine — "act like a media company"

  • Mechanic: a recurring, multi-format briefing (newsletter + podcast + short-form video + community post) whose hook is data only you have — "X% of our N users did Y yesterday." Add a monthly data-release for earned media (a legitimate editorial story, not an ad) and a rapid-response "news-jacking" cut tied to live events.
  • Job: create new intent rather than only harvest existing intent — reach the ~80% who aren't in-market today, build an owned audience you can re-market to at near-zero marginal cost, and lower long-run CAC as the audience compounds. First-party behavioural data is the one editorial angle competitors cannot copy.
  • Reach for it when: you sit on unique first-party behavioural data in a considered category, and performance CAC is rising as you compete for the same in-market pool.
  • Assembles: Newsletters (editorial) · Podcasts (owned) · YouTube Shorts · Digital PR / link earning · Research reports.

Lifecycle behavioural-trigger matrix

  • Mechanic: a precision (not broadcast) set of triggered sequences — each keyed to one behaviour or attribute (cash balance over $X · dormant >180 days · engaged-with-calculator), each with a defined target segment and an expected conversion outcome. Build it as a table: trigger → sequence → target → expected outcome.
  • Job: convert and cross-sell the existing base — the highest LTV:CAC opportunity in any programme, because these users have already cleared the trust hurdle. Each message is calibrated to where the customer actually is, not a generic blast.
  • Reach for it when: a multi-product, behaviourally-rich user base and a marketing-automation stack that can fire on events.
  • Assembles: Nurture / drip sequences · Usage-triggered campaigns · Renewal / upsell / cross-sell · Customer journey mapping.

Trojan-horse hero feature

  • Mechanic: lead a performance or brand push with one differentiating feature stated plainly — no feature list — chosen because most people want it or because it uniquely reframes the category. Make that single message the hero across your highest-reach placements.
  • Job: cut through clutter and own a distinctive wedge. A single, repeated hero claim out-performs a cluttered "we do everything" message and gives the brand a memorable entry point.
  • Reach for it when: a crowded category where competitors all run feature-list messaging.
  • Assembles: hero creative · Out-of-Home · Paid Search & Performance. Precedent: Apple "Shot on iPhone" (see library).

"Wrapped" shareable-data mechanic

  • Mechanic: an annual or periodic, personalised data recap engineered for organic sharing — each user's "year in review" (their best moment, their rank versus the community, their stats) rendered as a celebratory graphic with a built-in CTA.
  • Job: manufacture an organic-referral moment and a retention spike out of data you already hold. The user does the distribution because the artefact flatters them.
  • Reach for it when: you hold per-user data that can be framed as a status or progress story.
  • Assembles: AI-powered personalisation · UGC campaigns · Social — Organic. Precedent: Spotify Wrapped (see library).

Channel-as-JTBD — one creative idea per channel's job

  • Mechanic: before writing creative, define the audience's specific job-to-be-done on that channel, then design the idea and the "why it works" to that job — rather than reusing one execution everywhere. LinkedIn = social-signalling → a founder-led document ad; a rational-investor subreddit = validate the mathematically-correct path → a no-BS AMA with raw data; a sports portal = social currency → a pattern-interrupt that redirects the punter mindset.
  • Job: lift relevance and trust by matching the message to the mindset the channel puts people in, and reach a high-value audience via a premium/contextual channel where rivals aren't bidding.
  • Reach for it when: planning a multi-channel push — run it as a discipline, one row per channel (audience · JTBD · creative idea · why it works).
  • Assembles: any paid/owned channel rows, planned per-channel rather than once.

How agents use this library (updated 2026-06-12 — v3 architecture)

  • Creative Director — Phase 2 (Concept Design): draws from this menu when shaping each concept's channel mix and tactical mechanic; grounds §4b tactical precedents in named tactics. Tactics that don't earn their place don't go in a concept.
  • Campaign Manager — Phase 3 (Plan): composes the Plan's asset list from the selected concept's tactics — every chosen tactic becomes one or more asset rows (with review shape, copy file, owner, dependencies).
  • Campaign Manager — Phase 4a (Per-Step Brief): slices the relevant tactic definition(s) into Producer's brief. Producer does not load this file directly.
  • Operator: browsable HTML reference (marketing-tactics.html, mirrored at repo root as marketing_tactics.html) for planning conversations.

How this library is maintained

  • Operator-curated; updates are manual or session-driven (competitor teardowns, new-channel research). Worked examples live in tenant/library/ (added via /library-add) — that library holds what great looks like; this file holds the menu of what exists. Cross-link where useful.
  • Tenant-scoped by design: different industries and business models want different default menus. This master copy ships in the tenant kit and gets pruned/extended per tenant.
